Mobile Optimization for Checkout

A focused look at the mobile-first redesign of Scrapbook.com's checkout, improving accessibility, scalability, and user convenience on mobile devices.

Project Overview

This project aimed to optimize the checkout experience for mobile users. Key changes included making the layout responsive, using a 100vw container, eliminating pixel-based dimensions, and creating a scalable, accessible design that adjusts fluidly to different screen sizes.

Research & Problem Identification

BrowserStack testing and user feedback revealed issues with hard-coded pixel dimensions, a lack of scalability, and usability challenges on small screens. Addressing these issues required a complete overhaul of the layout to support flexible and responsive design principles.

Cross-device testing on BrowserStack showed layout inconsistencies and scaling issues, especially on smaller devices. This informed our decision to use a responsive, viewport-based design.

Users reported difficulty navigating the checkout page on mobile. The lack of a cart summary at the top and limited visibility of the checkout button led to frustration among power users.

Design Solutions

100vw Responsive Container

Converted the entire checkout process into a container set to 100vw, allowing it to scale naturally with the viewport width. This minimized reliance on media queries and fixed pixel sizes, making the layout fully adaptable.

Cart Summary and Quick Checkout Button

Added a cart summary and a quick checkout button at the top for power users. This streamlined the navigation experience, allowing users to proceed to checkout faster on mobile devices.

Responsive Flex Layout

Redesigned the checkout components with Flexbox, ensuring elements are arranged responsively without extensive media queries. This approach reduced file size and improved load time.

Viewport Meta Tag & Accessibility

Added the viewport meta tag to improve scaling and readability. Implemented accessibility improvements, such as larger touch targets, to meet WCAG standards and provide a seamless experience on small screens.

Prototyping & Testing

Using high-fidelity prototypes, we conducted tests on various devices to validate the fluidity and responsiveness of the checkout redesign. Iterative feedback led to minor adjustments in button placement and layout spacing for better usability on mobile screens.
